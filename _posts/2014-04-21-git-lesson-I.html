  1. 所有的git指令都被合并在git命令中，统一的方法是 git command args
  2. git init
  3. 三个config
    • 在版本库下执行：git config -e，配置当前版本库的配置
    • 运行git config --global -e，配置当前用户的所有版本库配置
    • 运行git config --system -e，配置所有用户的所有版本库的配置

  6. 三种diff
    • 比较工作区与暂存区：git diff
    • 比较工作区与版本库：git diff HEAD
    • 比较暂存区与版本库：git diff --cached
  7. git reflog 和 git log 的区别 
    git log最主要用于查看版本库的提交日志，一般用于从HEAD向前追溯，如果中间出现了reset，则不管；
而 git reflog 可以看到整个版本库的变化，commit 或 reset 均可以查看；
  8. reset和checkout的区别 
    checkout主要操作的是工作区，用于将工作区中的内容替换为版本库或暂存区中的指定版本；
而reset则主要操作暂存区或版本库，也可涉及到工作区；
  9. 危险的checkout
    • git checkout . 或 git checkout -- ：将工作区的内容用暂存区替换（会删除尚未提交到暂存区的更新，不会删除尚未加入到暂存区的文件，但会恢复已加入到暂存区但被从工作区域删除的文件）
    • git checkout HEAD . 或 git checkout HEAD ：类似上条，只是使用版本库
    • git clean -fd 删除工作区中所有未加入到暂存区的文件和目录；git clean -nd: dry-run
  10. 四种reset
    • git reset commit -- file：回滚从工作区到暂存区的提交，用版本库中的指定版本替换暂存区，但不会修改工作区，从这个角度来看，可用于add的反向
    • git reset [--mixed] commit：将暂存区回滚到指定的版本库版本，但不会修改工作区
    • git reset --soft commit：只调整版本库，不影响暂存区和工作区，主要用于回滚版本库到指定的版本
    • git reset --hard commit：同时调整版本库、暂存区和工作区
